# W像素主题商城 PRD 初版

## 一、项目定位

本项目只采用一个方案：`像素风主题商城`。

项目本质仍然是一个完整的 React 商城系统，不做脱离商城主线的趣味玩法，不做和课程要求关系不大的额外系统。所有新增内容都必须建立在商城业务基础上，服务于以下三个目标：

1. 完整覆盖 `第四次作业_大作业.pdf` 中要求的前台与后台功能
2. 在商城相关功能内做合理扩展，拉开与普通模板作业的差距
3. 最大限度展现前端能力，尤其是页面设计、交互细节、组件化、状态管理和前后台联动

整体风格已经确定为像素风，因此本项目将统一采用：

- 商品图像素风
- 首页视觉像素风
- 商品卡片像素风
- 按钮、边框、标签、弹窗像素风
- 后台管理端简化像素风

最终要呈现的不是“普通商城套一层滤镜”，而是“前台、交易链路、用户中心、后台管理都统一成像素主题的商城系统”。

## 三、项目主题定义

### 项目名称

Pixel Mall / 像素主题商城

### 项目概念

一个以像素美学为核心视觉语言的主题商城，售卖像素风或二次元气质的商品内容，例如鞋服、配饰、文具、家居小物、数码周边等。

系统由两部分组成：

- 前台用户商城
- 后台商品管理系统

这个项目的创新点不在于增加和商城无关的功能，而在于：

- 用统一的像素风视觉做出明显区别于普通作业模板的商城体验
- 在标准商城功能上加入更完整的商品展示、筛选、收藏、地址、规格、推荐等商城相关扩展
- 用更强的页面层次、状态反馈和组件复用来体现前端能力

## 四、视觉与设计方向

参考你们已有的“设计风格”图片，可以确定这套视觉方向不是传统街机像素风，而是更偏：

- 少女感
- 收藏卡片感
- 柔和粉色系
- 奶油色 / 杏色 / 粉色 / 金色混合
- 像素边框
- 像素商品图
- 带一点电商展示页的精致感

### 建议的整体视觉规范

- 主色：浅粉、珊瑚粉、奶油白、浅金
- 辅助色：薄荷绿、天空蓝，用于标签和点缀
- 卡片：像素描边 + 阴影 + 高亮角标
- 按钮：像素描边按钮、悬浮态高亮
- 标签：如“新品”“热卖”“限时”“推荐”做成像素贴纸感
- 图片：商品主图全部使用像素风素材或像素化处理后的图片
- 后台：保留简化版像素边框和统一色系，不做纯默认管理后台

### 美术执行原则

- 所有页面先统一一套色系和组件样式，再分头开发
- 先做公共组件视觉规范，再做页面
- 先确定商品卡、按钮、输入框、弹窗、导航栏，再铺开各页面

## 六、拓展功能

下面这些功能都严格属于商城相关功能，不是额外趣味功能。它们的作用是进一步体现前端能力，让项目从“完成作业”升级为“更像一个完整商城”。

### A 级拓展：强烈建议做

这些功能最值得做，因为既合理，又很能体现前端能力。

#### 1. 商品搜索

- 关键词实时搜索
- 搜索结果页展示
- 无结果空状态

这是首页搜索框最自然的落地，不应该只做一个摆设搜索框。

#### 2. 商品筛选与排序

- 按价格升序 / 降序
- 按热度排序
- 按新品排序
- 按分类筛选
- 按标签筛选

这类功能非常典型且电商化，能明显增加列表页的交互深度。

#### 3. 商品规格选择

- 尺码选择
- 颜色选择
- 款式选择
- 库存联动

这是商品详情页最有代表性的商城能力之一，也很适合做状态切换和按钮交互。

#### 4. 收藏功能

- 收藏商品
- 取消收藏
- 我的收藏列表

收藏既属于“我的页面”扩展，也有助于展示全局状态管理。

#### 5. 地址管理

- 新增地址
- 编辑地址
- 删除地址
- 默认地址切换

地址管理完全属于下单链路强化，是很标准的商城扩展。

#### 6. 浏览记录

- 最近浏览商品记录
- 我的页面中查看浏览记录

这类功能很商城，也很容易和详情页联动。

#### 7. 订单列表页

虽然 PDF 重点写的是订单详情，但建议你们把订单列表页一并做完整，因为这是商城主流程中非常自然的一部分。

建议支持：

- 待支付
- 已支付
- 已发货
- 已完成
- 点击进入订单详情

#### 8. 后台商品搜索与筛选

- 按商品名搜索
- 按分类筛选
- 按上架状态筛选

这样后台不只是最简单的表格 CRUD，而是更像真实管理端。

### B 级拓展：有时间再做

#### 1. 商品推荐

- 猜你喜欢
- 同类商品推荐
- 最近热门

#### 2. 支付倒计时

- 倒计时显示
- 超时未支付状态提示

#### 3. 优惠信息展示

- 满减提示

- 优惠券展示

不一定要做复杂结算逻辑，只做展示和页面组织也能增强商城感。

#### 4. 商品评价模块

- 评分展示
- 模拟评论列表

#### 5. 搜索历史与热门搜索

- 最近搜索记录
- 热门关键词展示

#### 6. 空状态页面

- 空购物车

## 十、5 人小组分工建议

你提到一个很关键的要求：`写代码最好是各自独立的页面，不要相互牵扯`。

这个要求非常合理，所以分工上应该采用：

- 先统一视觉规范与公共组件
- 再按页面归属拆分
- 每个人尽量负责一组相对独立的页面
- 只在少量公共组件和 service 层上进行协作

### 分工原则

1. 先有 1 人负责美工和素材规范
2. 页面按链路拆，不要多人同时改同一页
3. 公共组件先定样式，减少后续互相改来改去
4. 后台和前台尽量分开，减少冲突

### 推荐分工方案

| 成员 | 主要职责 | 负责页面 / 模块 | 说明 |
|------|----------|----------------|------|
| 成员 1 | 美工设计 + 公共样式 + 首页 | 像素素材生成、色系规范、商品卡样式、按钮样式、首页 | 先用豆包生成素材，先把整体风格定下来 |
| 成员 2 | 分类与商品展示 | 分类页、商品列表、搜索、筛选排序 | 负责商品展示链路前半段 |
| 成员 3 | 商品详情与交易链路 | 商品详情页、购物车页、创建订单页、支付页 | 负责最核心购买流程 |
| 成员 4 | 用户中心 | 登录注册、我的页面、收藏页、地址管理页、订单列表页、订单详情页 | 负责用户相关页面 |
| 成员 5 | 后台管理 | 后台登录、权限控制、商品管理页、新增编辑页、前后台商品联动 | 负责后台完整链路 |

### 这种分工的优点

- 成员 2、3、4、5 基本都能以页面为单位独立开发
- 成员 1 先做美工和公共样式，可以降低全组返工
- 页面边界清楚，不容易多人同时改一个文件
- 答辩时每个人都能清楚说明自己负责的部分

### 成员 1 的具体工作建议

因为你提到需要有人先去做美工，所以成员 1 不应该只是“帮忙配色”，而应该做更明确的前置工作：

#### 第一阶段：素材与风格规范

- 用豆包生成像素风商品图素材
- 用豆包生成像素风边框、按钮、贴纸、图标参考
- 确定主色、辅色、背景色、标签色
- 确定商品卡、按钮、输入框、弹窗的统一样式

#### 第二阶段：落地公共资源

- 整理素材命名
- 输出一份简短 UI 规范
- 完成公共 CSS 变量
- 完成公共基础组件样式

#### 第三阶段：页面实现

- 负责首页落地
- 协助其他人统一视觉

### 协作建议

为了减少相互牵扯，建议你们开发顺序如下：

#### 第一步

成员 1 先完成：

- 色系规范
- 素材方向
- 公共按钮
- 商品卡
- 输入框
- 页面基础布局

#### 第二步

成员 5 先完成：

- 商品数据结构
- 商品管理基础结构
- 前后台商品联动规则

#### 第三步

成员 2、3、4 分别开发自己的页面

- 成员 2：分类 / 列表 / 搜索 / 筛选
- 成员 3：详情 / 购物车 / 下单 / 支付
- 成员 4：登录 / 我的 / 收藏 / 地址 / 订单

#### 第四步

最后统一联调：

- 登录限制
- 收藏联动
- 购物车联动
- 订单联动
- 后台商品联动

## 十一、答辩展示建议

答辩时不要只说“我们做了哪些页面”，而要突出这三点：

### 1. 这是完整商城，不是静态展示页

- 前台购买链路完整
- 后台商品管理完整
- 数据能联动

### 2. 所有扩展都仍然是商城能力

- 搜索
- 筛选排序
- 规格选择
- 收藏
- 地址
- 浏览记录

这样老师会觉得你们是在“把商城做完整”，而不是“为了创新硬加奇怪功能”。

### 3. 前端能力体现明确

- 像素风统一 UI
- 多页面组织
- 组件复用
- 状态管理
- 交互反馈

---

## 十二、结论

你们现在最适合的路线非常明确：

- 只做 `像素主题商城` 这一套方案
- 所有新增功能都限定为 `商城相关功能`
- 用像素风统一商品图和整体 UI
- 用搜索、筛选、规格、收藏、地址、订单等功能来展现前端能力
- 用“页面独立分工 + 1 人先做美工和公共样式”的方式降低组员互相牵扯

如果要浓缩成一句话，可以这样定义你们的项目：

`一个以像素风为统一视觉语言、具备完整前后台商城功能，并通过商品搜索、筛选排序、规格选择、收藏、地址管理等电商扩展能力来重点展现前端开发能力的 React 主题商城。`
