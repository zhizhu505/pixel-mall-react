# Pixel Mall 后台像素风素材生成文档

本文件用于交接给素材生成同学。当前项目已经在 `src/assets/images/admin/` 下建立好目录和占位图文件，后续按本文档生成正式像素风 SVG 资源并替换对应占位文件即可。

## 1. 素材目录

```text
src/assets/images/admin/
  brand/       品牌与登录页素材
  bg/          背景、角标、分隔装饰
  dashboard/   后台首页统计图标
  empty/       空状态、加载、成功失败提示
  product/     商品占位图
  category/    分类图标
  order/       订单状态图标
  role/        角色权限图标
```

## 2. 替换规则

当前占位图统一使用 `.svg`，可以直接在浏览器打开预览。正式素材也统一交付 SVG，方便控制体积、保持硬边，并适配 Vite 的资源打包方式。

1. 直接用正式 SVG 覆盖同名占位文件。
2. 保持 viewBox、画布尺寸和文件名语义稳定。
3. SVG 内不要嵌入 base64 位图，避免文件体积失控。

必须保持文件语义命名，不要改成 `1.svg`、`icon.svg`、`new.svg` 这类名字。

## 3. 通用风格要求

整体风格参考：

```text
设计风格参考/mall2.jpg
Pixel Mall 首页视觉
```

关键词：

```text
奶油粉白、多巴胺、像素硬边、甜美促销感、粉黄薄荷蓝色块、少量深色像素描边、纯色错位阴影
```

统一色彩：

```text
深色描边：#3d2f3b
主粉色：#ff9fbd
亮粉色：#ff6fa8
奶油黄：#ffd765
薄荷绿：#7be0c3
天空蓝：#8fd7ff
奶油白：#fffaf0
柔粉底：#ffe2ec
危险红：#ff5f71
```

通用导出要求：

```text
背景：除 bg/admin-grid-tile.svg 外，其它图标和贴纸优先透明背景
边缘：硬边，不要模糊边缘
阴影：使用纯色错位阴影，不要模糊投影
描边：1-3px 深紫黑描边，关键轮廓清楚即可
文字：图片里尽量不要带中文或英文文字，避免缩小时糊
复杂度：后台图标要小而清楚，不要做成复杂插画
尺寸：按指定尺寸生成，允许 2x 高清导出
```

## 7. 每类资源的页面要求

### 后台首页

重点是 `dashboard/stat-*.svg` 六个统计图标。页面里统计卡本身已经是不同颜色的像素方块，所以图标不要太大，不要抢数字。建议图标主体占 64x64 画布的 70%-80%。

### 商品管理页

重点是 `product/product-placeholder.svg` 和 `empty/no-data-shop.svg`。商品管理页表格信息密度高，图片只用于占位和空状态，不建议在每行操作按钮旁额外加图。

### 分类管理页

重点是 `category/cat-*.svg`。每个分类图标色彩可以不同，但描边、颗粒大小、阴影方向必须统一。

### 订单管理页

重点是 `order/status-*.svg` 和 `empty/no-order.svg`。订单状态图标要小而清晰，最好能和状态标签配合，不要做成复杂插图。

### 角色权限页

重点是 `role/*.svg`。这些图标更像权限徽章，适合放在角色卡片标题旁或权限组标题旁。不要做人物大头像，优先做徽章、钥匙、菜单方块。

### 登录页

重点是 `brand/admin-logo-pixel.svg` 和 `brand/admin-login-hero.svg`。登录页可以比管理页更可爱，但表单仍然是主角，插画不要过度复杂。

通用模板：

```text
生成一张像素风 SVG 图标，透明背景，用于 Pixel Mall 奶油像素商城后台。
主题：[填写具体主题，例如 商品总数 / 待支付 / 分类数量 / 管理员徽章]

要求：透明背景，不要写文字，不要模糊阴影，不要 3D，不要写实，不要复杂渐变，不要嵌入位图，图形清晰，适合 [填写尺寸] 小尺寸展示。

```

## 9. 验收标准

生成素材交付前请检查：

```text
1. 文件名和本文档路径一致
2. 同组图标的尺寸一致
3. 图标缩小到 32x32 时仍然能看懂
4. 没有模糊阴影和写实质感
5. 背景透明的资源确实透明
6. 没有图片内文字
7. 深色描边不过重，色块足够明快
8. 与 mall2.jpg 的甜美促销感一致，但不会影响后台阅读
```
