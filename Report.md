## 2. 项目结构

本项目使用 React + Vite 开发，采用前台商城端与后台管理端分离的路由结构。页面、组件、业务 service、mock 数据、工具函数和样式文件按职责拆分，便于维护和扩展。

```
src/
├── main.jsx                         # 项目入口，挂载 Router、ThemeProvider、ServiceProvider
├── App.jsx                          # 根组件，通过 <Outlet /> 渲染子路由
├── router.jsx                       # 前台与后台总路由
├── router/
│   ├── h5Routes.jsx                 # 前台商城路由
│   └── h5LazyPages.js               # 前台页面懒加载入口
├── pages/                           # 前台页面与后台管理页面
│   ├── HomePage.jsx                 # 商城首页入口
│   ├── DetailPage.jsx               # 商品详情
│   ├── CartPage.jsx                 # 购物车
│   ├── CreateOrderPage.jsx          # 立即购买创建订单
│   ├── CheckoutPage.jsx             # 购物车结算
│   ├── PayPage.jsx                  # 模拟支付
│   ├── OrderListPage.jsx            # 前台订单列表
│   ├── OrderDetailPage.jsx          # 前台订单详情
│   └── admin/                       # 后台管理页面
├── components/
│   ├── h5/                          # 前台公共组件，如商品卡、搜索框、轮播、底部导航
│   ├── admin/                       # 后台公共组件，如后台框架、权限守卫、商品表单
│   └── common/                      # 通用按钮、弹窗、空状态、状态标签等
├── services/                        # 业务服务层，负责数据读写、业务规则和状态通知
├── api/mockApi.js                   # 统一 Mock API，模拟异步接口
├── contexts/                        # 全局 Context，如 ServiceContext、ThemeContext
├── hooks/                           # 自定义 Hook，如分页、无限滚动、service 订阅
├── mock/                            # 默认商品、分类、订单、用户、后台角色等 mock 数据
├── utils/                           # 工具函数及单元测试
└── styles/                          # 按 token、组件、页面、布局、响应式拆分的 CSS
```

## 3. 前台功能实现说明

### 3.1 商城首页

商城首页由 `HomePage` 和 `src/views/h5/Home.jsx` 实现，主要包含搜索框、活动轮播和热门商品展示。

- 搜索框使用 `SearchBar` 组件，用户输入关键词后跳转到搜索页，并通过 URL 参数保存搜索条件。
- 活动轮播使用 `Carousel` 组件，展示促销活动、新品上市、会员活动等内容。
- 热门商品列表使用 `ProductCard` 组件展示商品图片、名称、价格、原价、促销标签和已售数量。
- 首页商品列表使用 `usePagination` 做分页，每页展示固定数量商品，避免一次性展示过多内容。
- 用户登录后，商品卡会显示当前商品已加入购物车的数量，购物车数据来自 `cartService`。

### 3.2 分类页与搜索页

分类页用于按照商品分类浏览商品，搜索页用于根据关键词和筛选条件查找商品。

- 分类页左侧展示“全部分类”和后台维护的商品分类，分类名称通过 `splitCategoryLabel` 拆分为适合贴纸按钮展示的两行文字。
- 分类页右侧商品列表使用 `useInfiniteScroll` 实现滚动加载，减少初始渲染压力。
- 搜索页支持关键词、分类、折扣、库存和价格区间筛选，并支持销量、价格等排序。
- 商品列表只展示上架且有库存的商品，已下架或已删除商品不会出现在前台公开商品列表中。

### 3.3 商品详情页

商品详情页通过 `/detail/:goodId` 路由进入，页面根据商品 id 读取商品数据。

- 展示商品图片、详情图、商品名称、价格、原价、促销标签、库存、销量、分类和店铺卖点。
- 支持多规格商品选择，包括颜色、尺寸、包装、发货批次等规格组合。
- 根据用户选择的规格匹配 SKU 库存和价格，避免购买超库存商品。
- 支持“加入购物车”和“立即购买”。未登录用户会跳转到登录页，登录后返回原页面继续操作。
- 展示服务保障、优惠信息、详情说明、商品问答和商品评价，增强商品详情页完整度。
- 浏览商品详情时会记录用户足迹，便于在“足迹”页面查看浏览历史。

### 3.4 购物车

购物车页面通过 `cartService` 读取当前登录用户的购物车数据。

- 支持添加商品后合并相同商品和相同规格的购物车项。
- 支持单选、全选、数量增加、数量减少和删除购物车商品。
- 修改数量时会校验商品是否仍然上架，以及数量是否超过库存。
- 结算前调用 `validateCheckout` 检查是否已选择商品、商品是否可购买、库存是否充足。
- 购物车数据保存到 localStorage，刷新页面后仍可保留。

### 3.5 创建订单与结算

项目支持两种下单方式：商品详情页立即购买和购物车结算。

- 立即购买进入 `CreateOrderPage`，购物车结算进入 `CheckoutPage`。
- 下单页面展示商品清单、数量、规格、单价、总价和收货地址。
- 地址信息可从地址管理中选择，也可以在下单页直接填写。
- 下单时校验商品状态、库存、收货人、手机号和详细地址。
- 订单创建成功后会生成订单快照，保存当时的商品名称、价格、图片、规格和用户信息，避免后续商品修改影响历史订单。
- 订单创建后会扣减商品库存，并生成待支付状态订单。

### 3.6 支付流程

支付页通过 `/pay/:orderId` 进入，模拟真实支付流程。

- 页面展示订单号、商品摘要、订单金额、促销信息和订单状态。
- 提供 15 分钟支付倒计时，超时后禁止继续支付。
- 提供“像素钱包”“模拟银行卡”“奶油券”等支付方式切换。
- 使用模拟二维码区域展示订单尾号，模拟扫码支付体验。
- 用户可选择支付成功或支付失败。支付成功后订单状态变为已支付，购物车结算订单会清除对应购物车项；支付失败后可以重新支付。

### 3.7 订单列表与订单详情

订单功能由 `orderService` 统一管理。

- 订单列表按当前登录用户展示订单，并提供未支付、已支付、已发货、已完成等状态分类。
- 订单卡片展示订单编号、商品摘要、金额、状态和操作入口。
- 已发货订单支持用户确认收货，确认后订单状态变为已完成。
- 已完成订单支持评价商品，评价提交后会保存到订单数据中，并能在商品评价页展示。
- 已完成订单支持申请售后，包括仅退款和退货退款，并保存售后状态和物流信息。
- 订单详情页展示订单状态、商品清单、收货地址、物流时间线、评价记录和售后记录。

### 3.8 用户登录、注册与个人中心

前台用户模块由 `userService` 实现。

- 登录和注册页面支持用户名、密码、昵称等字段校验。
- 登录成功后保存当前用户到 localStorage，刷新页面后仍保持登录状态。
- 未登录访问购物车、下单、订单、个人中心、地址、收藏等页面时，会跳转到登录页。
- 个人中心展示用户昵称、订单入口、收藏、足迹、地址管理、消息中心和后台入口。

### 3.9 收藏、足迹、消息和地址管理

除基础购物流程外，前台还补充了多个商城常见功能。

- 收藏功能支持收藏和取消收藏商品，收藏列表可分页查看。
- 足迹功能记录用户浏览过的商品，便于回看。
- 消息中心展示系统消息和商品咨询消息，支持标记已读和聊天详情。
- 地址管理支持新增、编辑、删除、设为默认地址，并可从下单流程返回。

### 3.10 商品店铺与特色店铺

项目支持商品店铺展示和特色店铺专题页，用户可以通过店铺入口浏览店内精选商品。

- 商品详情页底部操作栏展示"店铺"快捷入口，点击后进入 `ShopPage` 店铺页。
- 店铺页通过 `/shop/:shopId` 路由进入，根据店铺 id 读取店铺数据，展示店铺封面、名称、标语、描述、亮点标签和概览统计（上架商品数、优惠进行中、入门价格）。
- 店铺页下方使用 `ProductCard` 组件展示店内精选商品网格，支持点击商品进入详情页。
- 店铺页提供"联系店铺客服"按钮，跳转到消息中心发起客服咨询。
- 首页通过 `FeaturedShopSection` 组件展示特色店铺卡片，点击进入 `FeaturedShopsPage` 特色店铺专题页，集中展示所有精选店铺及其商品数量和覆盖分类。

### 3.11 客服聊天

项目支持用户与店家客服的即时聊天功能，模拟真实电商咨询体验。

- 商品详情页底部操作栏提供"客服"快捷入口，点击后调用 `api.messages.openProductChat` 创建或打开该商品的聊天会话，并跳转到 `ChatPage` 聊天页。
- 聊天页通过 `/messages/chat/:messageId` 路由进入，展示会话标题、商品名称、消息概览（会话成员、消息条数、已读/未读状态）。
- 聊天页展示完整的聊天记录气泡列表，区分自己和对方的消息，支持自动滚动到最新消息。
- 用户可在输入框编写咨询内容并发送，发送成功后清空输入框；未登录用户点击客服会先跳转到登录页。
- 进入聊天页时会自动标记该消息为已读，通过 `message.markAsRead` 更新状态。
- 消息中心（`MessagesPage`）统一展示聊天会话和系统通知，支持按类型筛选、全部已读和未读计数。

### 3.12 商品问答（Q&A）查看

商品详情页内置"问大家"模块，支持查看和浏览其他用户的提问与回答。

- 商品详情页展示 Q&A 预览卡片，显示第一条问答摘要和总问答数量，点击后进入 `ProductQaPage` 问答页。
- 问答页通过 `/detail/:goodId/qa` 路由进入，通过 API 加载商品数据和问答列表。
- 问答页展示商品名称、问题总数、参与回答人数，并以问答卡片形式逐条展示每条问题的提问内容和官方/已购用户回答。
- 每条问答记录展示回答来源（如"已购买用户"）和参与人数，增强问答可信度。
- 问答页提供返回按钮可回到商品详情页，商品不存在时给出空状态提示。

### 3.13 商品评价查看

商品评价模块支持多维度筛选和详细评价内容展示，帮助买家了解商品真实反馈。

- 商品详情页展示评价入口卡片，显示平均评分、评价总数和分类统计（买家秀、追评、差评），点击后进入 `ProductReviewsPage` 评价页。
- 评价页通过 `/detail/:goodId/reviews` 路由进入，通过 API 并行加载商品数据和评价列表。
- 评价页顶部展示平均评分和评价总数，并提供多维度筛选标签：全部、买家秀（有图/视频）、有视频、追评、差评。
- 每条评价卡片展示用户头像（取昵称首字）、昵称、星级评分、规格信息、评价内容、媒体附件（图片/视频最多 6 张）、追评内容和商家回复。
- 差评评价会有特殊标记，商家回复以引用样式展示在对应评价下方。
- 评价页提供返回按钮可回到商品详情页，无匹配评价时给出空状态提示。

## 4. 后台管理端功能实现说明

### 4.1 后台登录

后台登录页面为独立路由 `/admin/login`，不与前台登录混用。

- 默认后台账号包含管理员和运营账号，数据来自 mock 数据并保存到 localStorage。
- 登录时通过 `adminService.login` 校验用户名、密码和角色配置。
- 登录成功后保存当前后台用户信息，进入 `/admin` 后台管理界面。
- 后台主路由通过 `RequireAdminAuth` 保护，未登录用户访问后台页面会跳转到后台登录页。
- 后台登录和前台用户登录使用不同 service，避免前后台身份混淆。

### 4.2 后台权限管理

后台权限系统由 `adminService`、`RequirePermission` 和 `AdminRolesPage` 共同实现。

- 项目内置管理员和运营两类角色。
- 管理员拥有 dashboard、分析、商品、分类、订单、角色权限等完整访问能力。
- 运营默认拥有 dashboard、分析、商品查看和订单查看等权限。
- 后台菜单根据当前角色的菜单权限动态展示。
- 后台页面通过 `RequirePermission` 做路由级权限控制，用户没有对应权限时无法访问该模块。
- 角色权限页支持调整角色可见菜单和具体权限，例如商品管理、折扣设置、分类管理、订单处理等。
- 修改角色权限后会保存到 localStorage，并记录后台操作动态。

### 4.3 商品管理

商品管理是后台管理端的核心模块之一，功能较完整。

- 支持按商品名称、描述、分类关键词搜索商品。
- 支持按分类和商品状态筛选，包括上架中、已下架、已删除等状态。
- 商品列表展示商品名称、分类、价格、原价、促销标签、库存、销量、状态和更新时间。
- 支持查看商品详情，包括商品描述、价格信息、发货说明、活动标签、优惠信息、服务保障、规格组、SKU 库存、详情文案和常见问答。
- 管理员可新增商品，填写商品名称、价格、分类、库存、描述、促销信息、服务保障、规格、SKU、详情模块和问答内容。
- 管理员可编辑已有商品，保存后会同步影响前台商品展示。
- 支持商品上下架。下架商品不会出现在前台公开商品列表中，也不能继续加入购物车或下单。
- 支持软删除商品。删除后商品保留历史订单快照，但前台不再显示。
- 折扣设置独立受权限控制，只有拥有 `products:discount` 权限的角色可以维护原价、现价和促销文案。
- 商品数据保存到 localStorage，因此刷新页面或重新进入后台仍能保留修改结果。

### 4.4 分类管理

分类管理用于维护前台分类页和商品表单中的分类数据。

- 分类列表展示分类名称、排序、描述和关联商品数量。
- 支持新增分类，新增后会出现在前台分类页和后台商品表单中。
- 支持编辑分类名称、描述和排序。
- 修改分类名称后，会同步更新该分类下商品的分类名称展示。
- 支持删除分类，但如果该分类下仍有关联的未删除商品，系统会阻止删除，避免商品出现无效分类。
- 分类数据持久化到 localStorage，刷新后仍保留后台维护结果。

### 4.5 订单管理

订单管理用于后台查看和处理前台产生的订单。

- 订单列表展示订单号、下单时间、用户、商品清单、订单金额、订单状态、评价数量、售后数量和物流信息。
- 支持按订单关键词搜索，可搜索订单号、用户、商品、评价内容和售后原因。
- 支持按订单状态筛选，包括未支付、已支付、已发货、已完成。
- 支持按评价情况和售后情况筛选，便于后台快速处理待回复评价和售后申请。
- 已支付订单支持后台发货，发货后订单状态变为已发货，并写入物流记录。
- 后台可在订单详情中回复买家评价，回复内容会保存到订单评价数据中。
- 后台可处理售后申请，包括同意、拒绝、确认收到退货、确认退款等操作。
- 售后处理会更新售后状态，并同步写入订单物流时间线，前台订单详情可以查看处理结果。

### 4.6 后台首页与分析中心

后台首页和分析中心用于展示商城经营概览。

- 后台首页展示商品数量、订单数量、待支付、已支付、已发货、已完成、低库存等统计数据。
- 分析中心展示经营数据、分类表现、库存风险和运营活动等信息。
- 后台操作动态记录管理员的登录、商品维护、订单发货、角色权限调整等行为。

## 5. 路由设计

项目采用 React Router 的 `createBrowserRouter` 配置多页面路由。前台商城与后台管理端挂在同一个根组件下，但分别使用不同的 Shell 和权限守卫。

```jsx
const router = createBrowserRouter([
  {
    path: '/',
    Component: App,
    children: [
      {
        Component: H5Shell,
        children: h5Routes,
      },
      {
        path: 'admin/login',
        Component: AdminLoginPage,
      },
      {
        path: 'admin',
        element: (
          <RequireAdminAuth>
            <AdminShell />
          </RequireAdminAuth>
        ),
        children: [
          { index: true, element: <RequirePermission permission="dashboard:view"><AdminDashboardPage /></RequirePermission> },
          { path: 'products', element: <RequirePermission permission="products:view"><AdminProductsPage /></RequirePermission> },
          { path: 'categories', element: <RequirePermission permission="categories:view"><AdminCategoriesPage /></RequirePermission> },
          { path: 'orders', element: <RequirePermission permission="orders:view"><AdminOrdersPage /></RequirePermission> },
          { path: 'roles', element: <RequirePermission permission="roles:view"><AdminRolesPage /></RequirePermission> },
        ],
      },
    ],
  },
]);
```

## 6. 状态管理与数据存储

- **全局状态管理方式：** 使用 Context + Service 模式。`ServiceProvider` 将商品、订单、购物车、用户、后台、地址、收藏、足迹、消息等 service 注入全局，页面通过 `useServices` 和 `useServiceSnapshot` 读取并订阅状态变化。
- **数据存储方式：** 使用 Mock API + localStorage。`mockApi.js` 统一模拟异步接口，service 层负责业务规则和数据持久化，例如商品、分类、购物车、订单、用户登录状态、后台登录状态、角色权限、地址等数据均保存到 localStorage。
- **前后台数据联动方式：** 前后台共用同一组 service 数据。后台新增、编辑、上下架或删除商品后，会同步影响前台商品列表和详情页；前台下单会扣减库存并生成订单，后台订单管理可以查看和处理这些订单；后台发货、评价回复和售后处理后，前台订单详情也会同步显示最新状态。

## 8. 遇到的问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 首页商品列表原本按两列瀑布流分组，平板/PC 下只修改 grid 列数无法真正变成三列或四列。 | 在平板和 PC 断点下对 `.pm-home-product-column` 使用 `display: contents`，让商品卡片直接参与外层网格布局，实现真正的多列展示。 |
| localStorage 中的商品、订单、购物车等数据需要前后台共享，同时还要避免页面刷新后丢失。 | 抽象 service 层统一管理业务数据，结合 `storage.js` 工具封装读取、保存、删除和 fallback 逻辑；前后台页面通过同一套 service 获取数据，实现联动和持久化。 |
| 项目需要补充单元测试，但不希望额外安装测试框架增加配置复杂度。 | 使用 Node.js 自带的 `node:test` 和 `node:assert/strict` 编写轻量单元测试，并在 `package.json` 中添加 `npm run test` 脚本，覆盖核心工具函数。 |
| 后台管理页面点击发货、评价回复、售后处理等操作按钮后，列表或弹窗中的订单状态没有自动刷新。 | 在操作处理函数（如 `handleShip`、`handleReplyReview`、`handleReturnAction`）中，API 调用成功后显式调用 `api.orders.reload()` 或 `api.products.reload()` 触发 service 层重新加载数据；同时通过 `useServiceSnapshot` 订阅 service 状态变化，确保 UI 自动同步最新数据。对于发货操作额外加了 reload 调用保证状态及时更新。 |
| 后台管理的详情弹窗（如商品详情、订单详情）中标签文字过长时会超出屏幕边界，影响阅读体验。 | 对 `.pm-admin-detail-grid` 使用 `grid-template-columns: repeat(2, minmax(0, 1fr))` 并配合子元素 `min-width: 0` 防止 grid 子项溢出；对 `.pm-admin-detail-span`（跨列的收货信息等长文本区域）设置 `grid-column: 1 / -1` 占满整行并使用 `white-space: pre-wrap` 允许换行；对表格内操作按钮区域使用 `flex-wrap: wrap` 防止按钮溢出。 |
| 项目需要补充单元测试，但不希望额外安装测试框架增加配置复杂度。 | 使用 Node.js 自带的 `node:test` 和 `node:assert/strict` 编写轻量单元测试，并在 `package.json` 中添加 `npm run test` 脚本，覆盖核心工具函数。 |
| 前台购物车、下单、支付、订单等多个环节共用同一份用户与商品数据，跨页面状态同步时容易出现数据不一致（如库存扣减后购物车未及时刷新）。 | 将购物车、商品、订单的读写操作统一收口到各自 service，页面通过 `useServiceSnapshot` 订阅 service 变更，保证任何一处数据修改后相关页面自动重新渲染，避免手动同步带来的遗漏。 |
